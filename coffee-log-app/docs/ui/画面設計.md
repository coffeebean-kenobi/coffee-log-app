# 画面設計

## 画面一覧

Coffee Log Appは以下の主要画面で構成されています。

| 画面名 | パス | 説明 |
|-------|-----|------|
| ホーム | / | アプリのメインページ、ダッシュボード |
| サインアップ | /signup | 新規ユーザー登録ページ |
| サインイン | /signin | ユーザーログインページ |
| プロフィール | /profile | ユーザープロフィール情報 |
| プロフィール編集 | /profile/edit | プロフィール情報編集 |
| コーヒー一覧 | /coffee | 登録済みコーヒーの一覧 |
| コーヒー詳細 | /coffee/[id] | 特定コーヒーの詳細情報 |
| コーヒー作成 | /coffee/new | 新規コーヒー登録 |
| コーヒー編集 | /coffee/[id]/edit | コーヒー情報編集 |
| テイスティングノート追加 | /coffee/[id]/tasting-note/new | 新規テイスティングノート登録 |

## レスポンシブデザイン対応

各画面は以下のようにレスポンシブ対応を行います：

### モバイル (sm: 640px以下)

- 単一カラムレイアウト
- ナビゲーションをハンバーガーメニューに変更
- フォーム要素は縦に並べる
- カードは1列表示

### タブレット (md: 768px以下)

- 2カラムレイアウトに対応
- カードは2列表示

### デスクトップ (lg: 1024px以上)

- マルチカラムレイアウト
- サイドバーナビゲーション表示
- カードは3列以上表示

## ユーザビリティ考慮事項

1. **直感的なナビゲーション**:
   - 明確なラベル付けと一貫したUI要素
   - ブレッドクラムによる現在位置の表示

2. **フォームデザイン**:
   - 必須項目の明確な表示（*）
   - インラインバリデーションメッセージ
   - 段階的フォーム（複雑なフォームを複数ステップに分割）

3. **フィードバック**:
   - アクション完了時のトースト通知
   - エラー発生時の明確なメッセージ

4. **パフォーマンス最適化**:
   - 画像の遅延読み込み
   - スケルトンローディング表示

5. **ダークモード対応**:
   - システム設定に基づくテーマ切り替え
   - 高コントラストモード
